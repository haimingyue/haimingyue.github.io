验证安全域名

网页授权域名是获取用户信息的回调域名。

微信授权流程

打开网页时候需要进行授权昵称。分享和支付之前必须要授权。

业务域名&js 接口安全域名&网页授权域名的区别

  • 业务域名
    • 当前业务使用的域名
    • 支付密码的时候，不会出现安全提示
  • js 接口安全域名
    • 微信开放的接口使用安全域名
  • 网页授权域名
    • 打开 app 的时候，使用微信授权登录。
    • 点击跳转微信的 app 内，然后回调到这个页面。配置回调页面必须是我们自己的页面。

网页授权的 access_token 和 普通的 access_token 的区别

  • 网页授权的 access_token，用来获取用户的信息，头像逆臣
  • 普通的：作用防范，调用非网页授权的所有接口

UnionID： 公众平台的唯一 ID（小程序、服务号统一 id）

使用场景：

每一个应用对一个用户分别对应 openid。但是 unionID 是唯一的。

授权流程

  • 用户统一授权
  • 通过 Code 换区 Access_token
  • 拉取用户信息

静默授权和普通授权的区别

  • 静默授权不需要用户同意，但是无法获取用户的信息
  • 用户同意授权。获取 code
  • 通过 code 获取网页授权的 access_token
  • 拿到 access_token 就可以拉去用户信息了。（scope：snsapi_userinfo）

JS-SDK 调用流程

  • 绑定域名：js 接口安全域名、业务域名…
  • 引用 js-sdk 文件
  • 通过 config 接口注入权限验证配置（先取到接口签名）
  • 通过 ready 接口处理成功验证

H5 如何接入微信分享

官方文档

第一步：请求路由的配置

export default {
  // api_test: '/api/test'
  // 跳转放在服务端
  wechatRedirect:
    "/wechat/redirect?url=http%3A%2F%2Fm.imooc.com%2Findex&scope=snsapi_userinfo",
  wechatConfig: "/wechat/jssdk", // 获取jssdk的接口
  getUserInfo: "/wechat/getUserInfo",
  payWallet: "/wechat/pay/payWallet",
};

第一步：判断是否登录

mounted() {
    this.checkUserAuth()
 },
   methods: {
    checkUserAuth() {
      let openId = this.$cookie.get('openId');
      if(!openId) {
        window.location.href=API.wechatRedirect;
      } else {
        this.getConfig()
      }
    }
  }

第二步：获取 config 信息, 然后执行 ready 方法注入分享功能。

getConfig() {
      this.$http.get(API.wechatConfig+'?url='+location.href.split('#')[0]).then(response => {
        let res = response.data;
        if(res.data === 0) {
          let data = res.data;
          // 注入config
          wx.config({
            debug: true, // 开启调试模式,调用的所有api的返回值会在客户端alert出来，若要查看传入的参数，可以在pc端打开，参数信息会通过log打出，仅在pc端时才会打印。
            appId: data.appId, // 必填，公众号的唯一标识
            timestamp: data.timestamp, // 必填，生成签名的时间戳
            nonceStr: data.nonceStr, // 必填，生成签名的随机串
            signature: data.signature,// 必填，签名
            jsApiList: data.jsApiList // 必填，需要使用的JS接口列表
          })
          wx.ready(() => {
            util.initShareInfo(wx)
          })
        }
      })
    }

注意：需要加上 url 进行签名（服务端完成）

第三步：分享资源的配置

//util.js
export default {
  initShareInfo(wx) {
    let shareInfo = {
      title: "灯塔", // 分享标题
      desc: "描述", // 分享描述
      link: "http://m.imooc.com/#/index", // 分享链接，该链接域名或路径必须与当前页面对应的公众号JS安全域名一致
      imgUrl: "", // 分享图标
    };
    wx.onMenuShareTimeline(shareInfo);
    wx.onMenuShareAppMessage(shareInfo);
    wx.onMenuShareQQ(shareInfo);
    wx.onMenuShareQZone(shareInfo);
    wx.updateAppMessageShareData(shareInfo);
    wx.updateTimelineShareData(shareInfo);
  },
};
